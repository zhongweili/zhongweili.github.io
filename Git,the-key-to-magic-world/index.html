Git，打开通向魔法世界大门的钥匙

发表于 2017-05-07

世界迟早由比特构成，程序员就是这个世界的建筑师。所以，当我看到人们担忧机器人作乱，却从来不担心程序员造反的时候，我都觉得人类奇妙极了。如果真有机器人统治世界的一天到来，在那一天之前，程序员应该早已经统治了世界许久。 ---比特仁波切和菜头《程序员、魔法师和麻瓜》

作为一名涉世未深的初级程序员，看到如此的论调是心潮澎湃的。这样的魔法世界在旁人读来是心向往之，而我竟然是有机会驰骋其中的！

的确，互联网的大潮汹涌澎湃，几乎所有人都已经离不开计算机、手机，不懂程序似乎就抓不住未来了，于是总听到有许多人说在自学编程，想要拿到那张船票。

程序确实是魔法世界的基石，但是会编程序可不意味着会施魔法。因为很显然，编程序并没有施魔法那么酷炫，做的最多的事情不是挥舞魔法棒，而是辛辛苦苦实现需求和绞尽脑汁修改bug。一方面，市场上的空缺引来了大批生造的程序员搬砖工，做着重复性的CRUD的机械式工作，就以为掌握了程序的奥义。另一方面，科班出身的大学生学习着最基本的计算机知识，迷失在页面排序算法的堆栈里进进出出。似乎都与印象中黑客级别的魔法师不怎么像。

遥想当年，阴差阳错进了软件这个专业，听着各种大师的故事，本以为等待我的会是星辰大海，谁成想竟是把眼看花的符号、搞不清方向的指针、没完没了的程序错误……正当我兴趣消磨之际，我看到一本教父级的Paul Graham写的《黑客与画家》，然而…并没有看懂。于是我扔掉书本又钻进了二叉树的随机森林里。

也许很多时候就是这样，彼岸的灯火闪着斑斓的光，可是去往彼岸的途中总是漫长的求索。这时候多么希望搭上一艘快船，让这求索之路变得欢快而轻便。

在外人看来程序的魔法世界璀璨夺目，想要进去只得翻山越岭，要是有任意门不就好了？其实互联网如此的开放，暗门处处都有，关键是要有一把对的上的钥匙。于我而言，Git就是我敲开程序大门的这把钥匙。

Git是什么？简而言之就是一个分布式版本控制软件。程序世界有数不尽的优美理论、简洁协议、优秀产品，为什么会是Git？因为它够奇妙。奇妙的东西一定有颠覆的地方，我第一次接触到Git的时候就被颠覆了，作为一个版本控制工具，Git做到了极致，只需敲几个命令，就可以坐上时光机穿梭在文件夹的各个版本间，见证历史片刻的每一个角落。那个时刻就是我的一个顿悟瞬间，原来硬盘里的空间不是一个被塞得越来越满的盒子，而是没有尽头可以任意变换的数字0101010101，原来不需要复杂的操作只要几行字就能发生神奇的事情，原来计算机可以听我发号施令精确无比地为我完成任务，那感觉就像是掌握了点石成金的魔法棒。

除了简洁有力，Git的另一个奇妙之处在于其分布式的特点。何为分布式？可以理解成一群分布在不同位置的计算机结成同盟协调运行的系统。具体到版本控制，那便是一群计算机一起合作维护文件的版本。我就是在那时理解了分布式，也才理解了互联网何以兼容并包而生机勃勃。从拥有聪明大脑和灵活双手的智人无法窥见现代社会的复杂和瑰丽，因为现代文明是属于整个人类群体的智慧。同样的，那时我明白了，程序世界是靠着开放与合作才变得愈发精彩，而促成程序员联合起来的一大工具就是Git。

为什么程序员打招呼的场所是在GitHub？为什么电影里的黑客总是在噼里啪啦敲击键盘？为什么疯狂的程序员会对GitHub连击数执迷不悟？只有进到魔法世界里身临其境才能解开这些为什么。当我掌握了Git，打开这扇暗门后，真的是进入了另一个世界。

我发现命令行是家常便饭，学会了Git，我可以在命令行里探索Gradle、Maven、Homebrew、npm、composer……简直可以把鼠标仍了。我发现GitHub有太多意想不到的好东西，这里可以找到有趣的应用、爬虫工具、课堂笔记、微信公众号编辑器、游戏脚本、甚至编程作业答案…的源码，原来程序员有了idea就该实现也能够实现。我发现好的程序可以互相借鉴，遇到问题可以共同解决，程序世界扎扎实实，人人都该创造也都在创造。

找到一把钥匙，就可以打开一扇门，可以进到一个房间，再找到下一把钥匙，打开下一扇门。这样由程序写成的发现之旅就正是发现程序世界的法门。我的学习之路就是由一条条连在一起的线交错而成。这样处处连接的线条，上面不是做不完的一个个任务和终点，而是一个接一个的节点，延展开来，伸向远处，没有尽头。所以我得以在魔法世界里进行我的探索和发现。

想要学习编程，不妨从Git开始。想要学习任何东西，不妨先找到一把钥匙。

另：再次拜读《黑客与画家》，果然好书！
