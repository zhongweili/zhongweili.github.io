异想

发表于 2015-03-29

学久了数学，不仅思考问题更加有逻辑性和严谨性，还会学到一种高度抽象的总结性思维方式。有时候觉得，数学分析、抽象代数、信息论等这些抽象的理论简直就是从上帝视角得来的，那些精秒的定义和定理让你感觉像是完全置身其外的造物主。

偶尔我也会脑洞大开，试着想要洞察出些许微光。

认知

在Wiki上，认知的解释是，个体思维进行信息处理的心理功能。我们无时无刻不在进行着认知活动，不管是眼前掠过的，耳边听闻的，还是脑海中飘过的，熟悉的会有个印象，陌生的会探个究竟。对于一个无生命的特定的实物（有生命有感情的太复杂），我们是怎样认知的呢？

在信息技术中有一个玄乎的论调，万物皆对象。这大概是因为面向对象方法是软件开发的主流，因此这里的对象指的是类的实例化，对象的抽象是类，类的具体化是对象，比如水杯是个类，而眼前我用的这个水杯是个对象。万物皆对象，我想说，对象皆向量。这个向量有方向有大小，而且是个高维（有限维）向量，维度高到无法想象其存在形式。那类呢，很简单，类是对象的抽象，就是向量的方向。

那么我们怎么去认知一个高维向量呢？将其投影到熟悉的低维空间。我们都知道，一个二维向量可以用坐标表示为(a,b)，那么(0,a)和(0,b)就是这个向量投影到x轴和y轴的结果，类似的，一个三维向量(a,b,c)可以投影到直线x轴上(a,0,0)，也可以投影到x-y平面上(a,b,0)（当然还可以投影到其他线或其他面上），可以看到投影过后得到的是向量局部的特征。好了，可以来举例子了，对一部手机这个对象，它当然包含了太多信息，一个人对它的认识一定是有限的，可能只是品牌、颜色、尺寸、相机像素、手机系统等方面，那就是投影到了品牌、颜色……这些元素作为单位向量构成的低维子空间，我想这大概就是一种认知过程。

评判

为什么同样的一个对象，不同人对其认识并不相同呢？因为每个人都有自己的评判系统。

  • 首先，投影变换是将整个向量空间映射到它的其中一个子空间，每个人投影到的子空间并不相同，还是那部手机，甲是普通的消费者，只知道品牌、颜色、尺寸这些显而易见的信息，而乙是资深的手机评测者，熟悉CPU内核、相机算法、系统优化度、品控把握……他们两个对这部手机投影的结果当然不一样。
  • 其次，即使投影到了相同的子空间，投影的方式很可能不一样，像上面举的简单例子都是正交投影，正交的概念是由向量内积引出的，简单的可以类比到垂直（内积为0），像太阳直射下来，显然大部分情况下阳光是斜射的且角度不同，故不同的投影方式也会导致结果不同，也就是不同人对同一维度信息的衡量不一样。
  • 进一步考虑，不同源于对比，而向量作为矢量是无法进行比较大小的，除非将其量化，其实人与人之间认识的差异实质上是对象在某一特定维度上的比较，虽然我们脑海中对象是完整的存在，可一旦描述它运用它的时候，我们使用的是对象在一个个坐标轴上的标量值。

幸运的是，在人们的共识下，有一个标量可以代表对象这个向量，那就是『价值』。给一个对象作为输入，就得到一个具体的标量值作为输出，即价值，这样一个函数我们并不清楚它的表达式，或者我们这里可以说成是『映射』。实际上，映射是更宏观的一种表述，函数是映射，投影也是映射，不妨大胆地想象，上面的这个映射可以理解成『价值观』（不知不觉间我们把实体对象的概念推广了）。

决策

人认知一个事物后，天性使人生出评判它的冲动，最终是为了进行决策。我们不妨认为，决策即选择，并且只要搞清楚两个选项的选择，多个选项的问题也就解决了。

看起来似乎很容易，只要将选项带入上文的价值观映射，得出结果比较大小即可。然而事实并非如此简单，因为每个人都不是孤立的，人的决策当然也是会受影响的。

虽然人人都认同用价值来评判事物，可是每个人得到的价值却不尽相同，要使价值可换算，须有一个一般的标准，这就是价值的表现形式『价格』。诚然，价格不是随意而定的，价格需要能够反映所有人的意愿，供需关系大概就是一种反映形式。具体是怎样我们不得而知，我们知道的是，从价值观到价格的这个映射（这实际上是函数的函数，就是泛函）给出了一种标准。

于是，人们依据标准进行对比决策。现实是，大部分的决策所需的计算量人脑是达不到的，人们往往同时靠理性和直觉来决策，而直觉常常是带有先验性的，所以对于决策有利也会有弊。

其实这些异想的只言片语在我脑海中存在了很久，只是在这儿试图将它们拼凑起来，希望能有一定的结构。我明白这些论调漏洞百出，根本没有根基，但我觉得这是启示性的，展开来可以有一大堆例子，我并没有都写出来，每个人肯定能有不一样的解读，只要经过了思考。

我喜欢的微信订阅号

发表于 2015-03-23

看到很多博主写了类似”我喜欢的Apps”这样的文章，我也想列出一些从我个人角度喜欢的Apps，毕竟Android平台可以瞎折腾，于是装了又卸，卸了又装，用过的应用大概有几百个吧，可是前段时间刚刷过系统只保留了最常用的几个，实在不想再翻出来去找链接找截图，只好作罢。

突然想到可以分享一下平时常用的微信订阅号，这两年微信订阅号数量经历了井喷式的增长，到处都有各种推荐和推广，也有平台专门统计出订阅号的数据给出排行榜，不过我只是想挑出几个对我有价值的介绍介绍，因为我确切地知道它们对我是有帮助的。

微信公众号，是微信区别于其他大部分移动社交应用包括国外应用的一个亮点，也是使微信从即时通讯工具上升到移动平台的一个重要功能。微信公众号又分为服务号、订阅号、企业号三类，因而微信上滋生出了各种客服平台、推广平台、微商户、自媒体、企业助手……一时间呱声一片，生机盎然。订阅号就是『自媒体』这个新生群体的主战场，每日可有一次推送机会送达用户的微信（跟腾讯关系好的貌似不止一次机会，appsolution就有两次）。其实订阅号实质上是一种移动端的RSS，订阅过后聚合推送更新，这是一种效率的信息筛选方式，用在移动端更是有效。

以下就是我的订阅号推荐，乱序。

PingWest中文网

微信号：wepingwest

有品好玩的科技，一切与你有关。这是一个科技资讯类的帐号，关键是，它让前沿、新锐和全球化的科技都与你相关。在这里基本可以纵览到新潮的互联网科技相关资讯，并且很多资讯都是带着个人化的观点表达的，甚至还有不同编辑对同一事件或产品的辩论，另外，也有一些文章是PingWest创始人骆轶航的访谈和思考，除了文章推送，这个公众号每周还会发布本周相关科技事件的测试题，非常好玩。内容方面，基本上每天会有三篇以上的文章，数量肯定足够让你了解科技新闻，更重要的是，文章很少有充斥科技圈的浮夸渲染的味道，质量也有保证。

MacTalk By 池建强

微信号：sagacity-mac

这个帐号开端于Mac技巧，最初记录一些作者使用Mac的技巧和心得，后来作者写了很多随笔，渐渐形成了一种技术与人文结合的风格，我也越发对作者有兴趣。作者池建强是一个70后程序员，然而他的文章几无暮气，他往往用最平实的语言讲述他遇到的事和想到的道理，毫不激进又能引人深思。读他的文章，单看语言，会有一种古风尚存的感觉，内容方面，纯的技术文章很少，实际上作者越来越多地开始谈生活，在这里，池老师不仅将技术与人文结合，而且很多思想都升华到生活上，我还读过他去年的书《MacTalk：人生元编程》，这里确实有着值得回味的东西。

LateNews by 小晚

微信号：LateNews

这是财经记者宋玮的公众号，里面的文章大部分是一些深度的事件解读和人物挖掘，往往篇幅较长，层层递进，是真正的深度评论，而且作者言语犀利，毫不避讳，有的文章遭到过封查。我尤其喜欢里面的一些人物品评的文章，认真读过后，对马云、马化腾、周鸿祎、雷军、傅盛等等人物有了进一步的认识，他们并不一直是台前那样的光鲜，他们也是一样的人，作者的文章让我能够体会到这些大人物的真实，其实也还能了解到这些人物之间的纠葛和八卦，还会从另一角度对他们控制的企业公司有所认识，还是挺有意思的。

改变自己

微信号：wechanger

一起来，每天改变一点点。这是这个帐号的slogan，我刚开始以为这会是个每天灌鸡汤的地方，可是加了之后发现，这里真的是满满的干货！后来我又想，即使这里教你如何积极上进的方法，可是这个主题怎么可能一直持续下去呢？然而这里真的能让人每天都有改变。原来这里已经形成了较成熟的社群，有相对完善的会员制度，会员内部还有分散的改变会，文章多为由志愿者精心整理的外文翻译，还时不时有各种会员非会员的小活动，积极的氛围非常浓厚。具体的内容我就不介绍了，总之这个公众号绝对价值足足的。

小道消息

微信号：WebNotes

这是丁香园创始人冯大辉的公众号，冯大辉其人，早年任职于阿里巴巴，后出来创业，创立了丁香园且接受了腾讯的巨额投资，可谓是风云人物，评价褒贬不一。他本人很有个性，文章往往带有个人色彩，文章的评论里也经常出言不逊，而他的文章又很有思考的余地，总是大胆地指出一些别人不愿提及的问题，还会有一些有趣又发人深省的东西。他在公号介绍里说有一句玩笑话，只有小道消息才能拯救中国互联网，只有小道消息才能拯救中国创业者。不知道这些大胆的小道消息有没有那么大作用，不过这些东西真的是别的地方看不到的。

超级数学建模

微信号：supermodeling

当初做数学建模时关注了这个公众号，虽然对比赛一点帮助都没有，但是却更加激起了我对数学趣题的兴趣。这个公号的运营者是广东几个大学的一些大学生，他们自发的组织起来，希望给所有数学爱好者提供一个交流分享的平台，看了他们的网站，愿景挺好的。文章大多是数学爱好者的投稿，涉及到的问题非常广泛，用数学知识来解释许许多多奇怪的现象，非常有趣。

世相

微信号：thefair

刚关注这个公众号时，看到头像着实一惊，不知这个赤裸的女人要在这里揭露世间的什么真相，可能是什么坏坏的东西吧，反正是好奇。读过一些推送的文章后，我发现这真是个有情怀的帐号，无论原创的还是转载的，都能看得出精心的编辑。世相已有四百余篇文章，这里谈刻薄时代的宽厚，谈普通生活的伟大体验，谈被背叛的人的力量，谈不能随口而说的心动……真的如它的介绍所说，世相倡导有物质基础的精神生活，文章兼顾见识与审美。

鬼脚七

微信号：taobaoguijiaoqi

我只能说，这个公众号的主人真是个有思想的人。淘宝鬼脚七这个电商领域的专家，在事业的辉煌的时候选择从阿里巴巴离职，他去搞自媒体、去禅修、去写作、去研究互联网，只为了做他自己。他从来不去迎合，却有无数赞赏，他从来不去修饰，却总道出真意。文章里写的最多的是对《老子》的解读，往往将朴实的道理透过生活中的小事折射出来。最让我佩服的是他对生活的态度，越了解他越觉得，他的生活那是褪去芳华后的本真。

TOMsInsight

微信号：TOMsInsight

这又是一个充满干货的公众号，专注于中国互联网的深度数据分析洞察，发布独家原创报告。怎样的分析算是深度分析呢？就是让你看到海底那十分之九的冰山。在这里，你认识的是一个陌生而真实的互联网世界。这个公号主要对互联网黑市进行分析，深度讲解了百度竞价单页、安卓地下渠道、网络彩票内幕、、攻击敲诈勒索、虚假运营手段等惊天黑幕，文章不多但篇篇是良心之作，绝对是深入洞察中国互联网的利器。

Octopress部署到gitcafe

发表于 2015-03-19

博客一直用着github-pages免费服务，虽时有不稳定的情况打开博客会慢一些，不过倒也能忍受，偶然用手机登录竟发现完全进不去。猛然意识到，因为我在学校里用的是教育网，才有此殊遇，若用拨号上网或是手机流量的蜂窝网络，则根本无法访问，所以事实是，github的pages服务被墙了。

偶然的机会，发现了Gitcafe，这是国内仿Github的一个项目托管平台，也提供团队协作方案和企业服务。网站的About下写着：We are building two bridges to connect people, one is from college and middle school students to real world projects, the other one is from China to western countries and vice versa.还挺有情怀的。同样是基于版本控制系统Git，让Gitcafe在国内几乎可以替代Github，于是我决定将博客同时部署到Gitcafe平台上。

原理

几经辗转，在Gitcafe上把项目建了又删，删了又建，终于摸清楚了原委。博客框架是Octopress的开源项目，将其clone到本地，即可自行编码改动及添如文章，github上建立的项目作为本地Octopress工程的remote源，可以进行版本存储和控制，而同时又是一个github-pages项目，可以将项目文件生成网页发布到设定的域名下，这就是博客的建立和部署及控制过程。

现在要想将博客同时部署在gitcafe提供的域名下，最方便的做法是，新建一个gitcafe-pages项目也作为本地Octopress项目的remote源，而并不对本地项目的源文件进行版本管理，只控制生成网页及发布的部分。这样，每次完成了新的博文，在github上记录源文件的保存，再更新网页发布部分，同时在gitcafe上也更新网页发布部分，那么就完成了在两个页面下的部署。

步骤

  1. 在gitcafe注册帐号，到公钥管理下添加自己的ssh公钥，这样每次提交就不用输入密码了。
  2. 新建与账户名同名的项目，gitcafe会将用户名同名项目默认为gitcafe-pages项目。

  3. 设置pages服务，网站又提供帮助文档，但是不要完全按照上面的步骤操作，不需要创建本地git目录和分支，因为Octopress就是本地项目，/_deploy目录下又master分支，所以只需进入/_deploy目录添加remote即可

    git remote add gitcafe git@gitcafe.com:zhongweili/zhongweili.git >> /dev/null 2>&1

  4. 随便进行一点修改进行提交，完成后就能访问xxx.gitcafe.io了。

说明

  • Octopress项目在github上有两个分支，source分支是在主目录下，控制源文件，master分支是在/_deploy目录下，控制pages服务的网页发布。上面的步骤中是将master分支remote到gitcafe上，并命名为gitcafe分支。
  • 每次提交网站内容时，要执行rake deploy，实际上是执行的Rakefile文件里的一段脚本内容,具体是进入/_deploy目录进行deploy并提交，只要在这里提交时一并提交到gitcafe上就可以了。

  • 为了不在每次提交博文时都写代码进行remote，只要将代码写入Rakefile里面就行了。

    cd "#{deploy_dir}" do
    system "git add -A"
    message = "Site updated at #{Time.now.utc}"
    system "git commit -m \"#{message}\""
    puts "\n## Pushing generated #{deploy_dir} website"
    Bundler.with_clean_env { system "git push origin #{deploy_branch}" }
    puts "\n## Github Pages deploy complete"
    system "git remote add gitcafe git@gitcafe.com:zhongweili/zhongweili.git >> /dev/null 2>&1"
    system "git push -u gitcafe master:gitcafe-pages"
    puts "\n## Gitcafe Pages deploy complete"
end

  • 如果有自己的域名，同时在github和gitcafe上设置域名的解析，可以达到访问你的域名自动跳转到github或gitcafe上的目的。

我发现国外有一些提供很有趣的服务的网站，比如gravatar的大头贴服务，disqus的评论系统，IFTTT的串联信息，恰到好处地展现互联网的连接，我有时候想那大概是在自由创新的互联网氛围下产生的，可惜的是在国内大部分都不能正常使用。国内有很多这样的模仿者，像模仿disqus的多说，模仿github的gitcafe等，虽然功能齐全，但是却似乎发现商业模式并不一样，都为适应国内的环境而有变化，因为面临着异常激烈的竞争和盈利压力，考虑的从来都是实用性，很难形成国外网站那种社区氛围。当然，国内的互联网企业也渐渐形成了自己的竞争力……这个问题先按下不表了。

有尽亦无穷

发表于 2015-03-15

电影《疑犯追踪》里Finch有一段话：

“π，圆周长与其直径之比，这是开始。后面一直有，无穷无尽。永不重复。就是说在这串数字中，包含每种可能的组合。你的生日，储物柜密码，你的社保号码，都在其中某处。如果把这些数字转换为字母，就能得到所有的单词，无数种组合。你婴儿时发出的第一个音节，你心上人的名字，你一辈子从始至终的故事，我们做过或说过的每件事，宇宙中所有无限的可能，都在这个简单的圆中。用这些信息做什么，它有什么用，取决于你们。”

真的是这样吗？听起来很精妙，还有点感动，可事实上并不能肯定，在数学上，满足这样性质的数字叫做正规数，而π至今未能证明是否是正规数。

如果是呢？那就是说任何信息都存在于一个数中！所有的信息只是数字的不同排列组合而已。能想明白但是难以接受，那转换到具体的实物吧，我们来考虑质量（在比较宏观的宇宙观中，质量才是永恒的，甚至时间都不是），也许，某一个物体只不过是一些原子的一种排列组合罢了，如果假定原子是最小单位的话，那么对于尺度更微小的物体呢，大概是普朗克常量的排列组合？

不如我们来看一个具体的例子，对于任一张1080p的图像，其为1920*1080像素，也就是可用一个行宽1920列宽1080的矩阵表示，一共包含2073600≈2.1M个像素点，而人眼能够识别大约一千万种颜色，则每个像素点的值有10M种可能性，那么一共存在10M的2.1M次方种不重样的静态图像。不出所料，图像的数量是有限的。将一帧一帧的图像拼接起来，再添上音频的数据，不就成了电影了吗？我们生活的场景不就是一个个电影吗！

如你所见，我们越来越接近这样一个事实，这世界是有限的。想象，今天眼前的一个画面、耳边的一段音乐、脑海的一段文字，可能在过去或者未来的某个时刻有着一模一样的一个copy，因为总有穷尽的时候，如果有足够的时间和资源，理论上可以造出所有的一切。可是，事实上并非如此，为何这么多人的繁衍交替中从没有完全相同的两张面孔？为何音乐作品、文学作品能持久地创新更替？

因为时间和资源不可能足够。有一个有趣的例子，一副扑克牌共54张，一共有54的阶乘54！种组合方式，这个数大约是10的71次方，这个数字的数量级已经接近于整个宇宙的基本粒子总数了（你应该从棋盘放米粒的故事中就认识到指数的可怕）。可以说，几乎每一次随机性的洗牌，都创造出了一种新的排列方式。因此完全不同担心无意间会造成重复。

那么，看起来是这样的，世界在我们的量化中是有限的，只不过，这个有限的值实在大得可怕，甚至可以直接当做是无穷的了。而这样的想法，实际上是在用有尽度无穷，难免会陷入永远追不上乌龟的那个芝诺悖论。要搞清楚这层关系，首先来理解什么是无穷吧。

有一道流传很广的趣题，无限循环小数0.999…是否等于1？答案是肯定的，看到流传着很多灵巧而浅显的证明，通过一些代数变形看似巧妙地推出结论，比如这样：

记 x=0.999...
则 10x=9.999...
那么 9x=10x-x=9.999...-0.999...=9,
故 x=1.

或者这样：

由 1/3=0.333...,
则 1=(1/3)×3=0.333...×3=0.999...

但是这些方法并不严格，只是形式上的直观解释。原因是无限循环小数的四则运算并不是想当然认为的这个样子，事实上无限小数的表示是超出了初等数学的范围的，要想严格地证明，需要理解无限小数背后的有理数、无理数的严格定义，这要用到数学分析中的Dedekind分割.

可以看到，并没有那么容易。其实这背后暗含着一个道理，对于新事物的研究，不能用对待旧事物的方法体系去理解。数学上的几次危机如根号2的出现、复数的出现、非欧几何的出现等公理化体系的崩塌都是由于新的体系在演替，也正因此数学才能历久弥新。

现在，我们可以肯定的是0.999…等于1，关键是怎么理解。看起来似乎0.999…总是比1小那么一点，虽然一直在不断地逼近，然而这种逼近又是没有止尽的。正如高等数学中最基本的概念极限，就是一个无限逼近的过程。对，过程！其实无穷不仅是个结果，也是个过程。分形就是体现数学之美的过程，周而复始，无穷无尽。

现实中无穷的过程也可寻见，譬如著名的英国海岸线长度问题，离的更近总能发现更多长度的细节，以至于难以选择恰当的尺度进行测量。就像分形一样，事物无穷尽地细分下去，或许还存在着完全相同的结构，只不过超出了我们的观察范围罢了。

想象一下，我们所了解的这个世界只是其一而已，从空间角度看，微观尺度下的粒子就有其世界，不过我们无法观测到，从维度角度看，一个二维的面就是个世界，也许展开成三维形式跟地球无异，不过我们无法探求到。谈到世界，还有一种平行世界/平行宇宙论,恐怕在科幻作品里比比皆是，这个假说可以解释时光旅行等很多奇妙的事。

也许，所有这些都是解释。人类为了描述世界，因而将其量化，于是一切皆有解，又为了认识世界，因而不断探其本源，于是无穷得以生。这世界，有尽亦无穷。

信息隐藏之数字水印

发表于 2015-02-28

信息隐藏技术因其不可感知的特性一直都有种神秘感。古希腊战争中，为了安全传送军事情报，奴隶主剃光奴隶的头发，将头发缝在其头皮上，待头发长起来后再派出去传送消息，这就是一种信息隐藏，再比如古代的藏头诗、藏尾诗等，都是将隐秘信息用巧妙的方法藏起来使外人无从知晓。对于携带保密信息的载体，其加密技术并不一定多么高明，然而未授权的人看到载体根本不知道保密信息存在与否！这就是信息隐藏的魅力所在。

信息隐藏指在设计和确定模块时，使得一个模块内包含的特定信息（过程或数据），对于不需要这些信息的其他模块来说，是不可访问的。

现代社会，计算机已经深入到人们生活的方方面面，而在计算机上一切信息都是由0/1数字组成的，文本、图片、音频、视频，在计算机的世界里可以将它们拆解成同样的本元再用各种方法将它们重新组合，因此，这样想来可以玩出许许多多有趣的信息隐藏的伎俩。目前，信息隐藏技术主要的用途在于数字作品的版权保护方面，也就是在数字作品中嵌入代表版权的数字水印，并且这种水印是肉眼不可见的。本文将讲解一种在图片文件中嵌入水印的方法。

图像的表示

世界的一切都是数字组成的

忘记哪位哲人说过这句话，这似乎在揭示数字背后深不可测的涵义，但我不想究其深意，我觉得从描述世界的角度来看，一切事物都是需要用数字来度量的。在计算机的世界中，不存在模糊不存在无穷，所有的东西都是数字来表示和计算。对于一张灰度图片，用分辨率表示图片的长宽，用像素表示其明暗，以二维矩阵的大小度量分辨率，以矩阵元素的值度量像素值，如此，则将一个灰度图像与一个二维矩阵相对应起来。（彩色图片又是怎样的呢？固定大小的矩阵又如何表示无限多的图片呢？）

那么接下来就是对这个矩阵中的数字做手脚了，在矩阵中藏进一些新的数字，而使对应的图片与原来的图片看起来一样。

嵌入水印

怎样在一个铁盒中藏下一个铁块？将铁块熔于铁盒。

我们的水印不管是文本还是图片，它都可以提取成一串数字，怎样将这串数字熔于载体图像的那个矩阵中呢？当然了将这串数字与矩阵进行简单的四则运算就可以消失得无影无踪，但我们总要把水印再提取出来的，并且还要求这个图片能抵御压缩、剪切、变换等攻击，这样看来粗暴的『物理反应』并不能达到目的，或许需要来点『化学反应』，对，就像把金属藏于金属化合物一样！

其实，一个图片有两种展现形式，一种是在空间域，就是我们看到的样子，由不同位置深浅不一的点构成，另一种是在频率域，由不同频率的强弱各异的波构成。

就像同一事物用不同的两种语言表达，这里的傅里叶变换充当的就是一种翻译工具而已，也可能，在这种语言里增加一个词对应的意思在另一种语言的表达里可能完全没变，我们要利用的就是这一点！

程序实现与实验

算法实现的思路为，利用分块DCT变换，水印信息按位嵌入一个个DCT系数块，通过调整两个中频DCT系数的相对大小就达到了隐藏一位密文信息的目的。水印的提取只要用逆过程就可完成。

关键代码行：

T=dctmtx(8);
DCTrgb=blkproc(data,[8 8],'P1*x*P2',T,T');
[row,col]=size(DCTrgb);
row=floor(row/8);
col=floor(col/8);
a=zeros([row col]);
[k1,k2]=randinterval(a,count,key);
for i=1:count
      k1(1,i)=(k1(1,i)-1)*8+1;
    k2(1,i)=(k2(1,i)-1)*8+1;
end
for i=1:count
if msg(i,1)==0
    if DCTrgb(k1(i)+4,k2(i)+1)>DCTrgb(k1(i)+3,k2(i)+2)
        temp=DCTrgb(k1(i)+4,k2(i)+1);
        DCTrgb(k1(i)+4,k2(i)+1)=DCTrgb(k1(i)+3,k2(i)+2);
        DCTrgb(k1(i)+3,k2(i)+2)=temp;
    end
else
     if DCTrgb(k1(i)+4,k2(i)+1)<DCTrgb(k1(i)+3,k2(i)+2)
        temp=DCTrgb(k1(i)+4,k2(i)+1);
        DCTrgb(k1(i)+4,k2(i)+1)=DCTrgb(k1(i)+3,k2(i)+2);
        DCTrgb(k1(i)+3,k2(i)+2)=temp;
    end
end
if DCTrgb(k1(i)+4,k2(i)+1)>DCTrgb(k1(i)+3,k2(i)+2)
    DCTrgb(k1(i)+3,k2(i)+2)=DCTrgb(k1(i)+3,k2(i)+2)-alpha;
else
    DCTrgb(k1(i)+4,k2(i)+1)=DCTrgb(k1(i)+4,k2(i)+1)-alpha;
end
end
data=blkproc(DCTrgb,[8 8],'P1*x*P2',T',T);
