种一棵树

发表于 2017-11-01

种一棵树，最好的时间是十年前，其次是现在。学编程亦如此。

我决定新学一门编程语言，一方面从零学起可以补漏一些计算机基础知识，另一方面为以后的工作储备，多一点发展方向。

为什么选择学习Go？左耳朵耗子的这篇文章讲得很精彩，他以一个前辈的视野论述了Go语言的优缺点和发展前景。

一个技术能不能发展起来，关键看三点：

  • 有没有一个比较好的社区
  • 有没有一个工业化的标准
  • 有没有一个或多个杀手级应用

用这些标尺来量一下Go语言，可以清楚看到：

  • Go 语言容易上手
  • Go 语言解决了并发编程和写底层应用开发效率的痛点
  • Go 语言有Google这个世界一流的技术公司在后面
  • Go 语言的杀手级应用是Docker，而Docker的生态圈在这几年完全爆棚了

好了，话不多说，本文记录一下Go环境配置。

Go安装

Windows

win10系统直接官网下载安装，环境变量都自动配置好了

Linux（CentOS）

下载

wget https://storage.googleapis.com/golang/go1.9.linux-amd64.tar.gz

解压

tar -C /usr/local -xzf go$VERSION.$OS-$ARCH.tar.gz

设置环境变量，~/.bashrc、~/.zshrc

Go命令

go build

编译

go clean

清除编译文件

go get

获取远程包（其实是两步，下载，go install）

go install

加载（生成可执行结果，并自动放到$GOBIN下）

go test

读取*_test.go文件，生成并运行测试用可执行文件

go run

编译并运行Go程序

代码示例

mkdir $GOPATH/hello

go install hello

hello

输出”Hello，world.”

项目开发中的反常识思考

发表于 2017-10-26

这学期集中精力做一个网站开发的横向项目，自知没有专业团队的水准，很多地方也不符合业内规范标准的方法，不过在这个过程中不断摸索着，具体的开发方法、处理问题的思路、团队成员的合作、进度的把握等等各方面都有所接触，总是有些体会的。趁着项目暂停的机会，做一些反思总结。

我回忆起日常工作中每一次决策的思路、实施的过程、造成的后果，力图从中挖掘出最核心最有价值的内容——反常识。 反思的关键在于否定，否定了旧有的认知，才能在自己的脑海中建立起新的结构。 每发现一处反常识，我都在这种颠覆中印象深刻。

1

常识：开发中学习新东西时，先掌握了所有知识，再进行开发

反常识：开发中学习新东西时，先动手去做，再回来学习知识

新手入门开发时，常常会有一个误区，先把所有的理论、方法、特点都弄懂，才开始动手做。比如要学习前端，听说是用 HTML，于是百度一下：HTML 是一种超文本标记语言……每一个字都认识，但是不知道在说什么，要想弄明白需要耗费巨额的时间。这就好比学游泳，不需要知道水有什么原理，只需要知道能用它做什么。因此，最好的入门方法，就是完全彻底地模仿最好的。

不要试图掌握所有知识，另一个重要的原因是，新的知识和方法在不断涌现，尤其在互联网发展迅猛的今天，每隔几个月就出现个新的框架，之前在旧有体系中的深挖意义不大。

从这一点来看，只是模仿别人动手去做是不够的，还需要回来继续学习知识。一位船长说他之所以能够通过险滩并不是因为知道所有暗礁的位置，而是知道一条安全的路线罢了。但是在软件开发中没有永远正确的路线，模仿别人的方法可能不是最安全的或者最有效率的，动手做好后要继续学习，找到适应场景的最合适路线。

2

常识：项目规划时，先定需求，再排工期

反常识：项目规划时，先定工期，再调整需求

项目规划，是为了保证项目开发能够按部就班地高效推进，但是真正做过了项目才发现，开发根本不可能按部就班，因为项目总是在变化中。为了追求效率，在进行计划安排时，要先定好工期，再基于工期去优化产品的设计和开发，决定取舍，取舍时自然会保留住核心需求和功能，过滤掉不影响结果的需求。毕竟，互联网时代，时间成本是最大的成本。

具体实施的时候，就要看技术的取舍之道了。对于给定的一个需求，常常只是一个简单的说明或者设计图，其中隐含的一些边界条件并不明确，而边界条件对技术设计的复杂度和开发难度影响极大。所以，开发时应该明白应用场景的边界在哪里，有针对性地选择，才能使实现方案变得简单，也就节省了时间降低了成本。

3

常识：提高开发效率，需要专心致志做手头的事情

反常识：提高开发效率，需要适当“三心二意”打破边界

专心致志认真工作当然没有错，错的是仅仅专心于分配给自己的正在做的任务。在软件开发中，程序员很容易陷进一个坑里原地打转，有时候是在一个无关紧要的地方钻牛角尖，有的时候则是在缺少资源的情况下难以前进，而那些资源可能只需要稍微抬头就能找到。小团队更需要注意，需求变动很频繁，可能在专心钻研的这项任务并不是核心，或者很可能马上会变动。

其实提高开发效率并不是提高个人写代码的速度，而是整个项目平稳推进的速度。打破任务和职责的边界，为了项目和产品整体考虑，才能够在团队中发挥更大的作用。适当的“三心二意”，关注一些队友的进展、不同任务的联系、需求的变更情况……就能更加精准定位真正的问题。

希望对你能有启发。

区块链：我们尚未准备好迎接的革新

发表于 2017-07-17

想象一下，你不再需要评估与你交易的陌生人的信用；想象一下，你不再需要丝毫担心存放你钱财的银行的信誉；你不再需要确信你国政府知道什么是公平公正的。

这将会发生什么？

改变世界。

而这，正是区块链的愿景。

加密货币，这个基于区块链的数字货币，最近因为 Mt.Gox 和 The Silk Road 的大跌而在新闻界成了一系列耸人听闻故事的主谋。

但是，实际上还有一些比数字货币更大的事情正在发生。

当主流媒体们还在忙着猜测变化无常的价格和挖掘黑市阴谋的时候，他们却错过了这样一个事实：密码学家们已经默默地发明了一套全新的技术原语。

区块链（以及支持它的相关协议）的发明源自于开发者试图解决的一个难题：如何创造一种数字化的、难以追踪的金钱。通过与密码学、博弈论、经济学和计算机科学的结合，他们设法创造出了一整套用于建立分布式系统的工具。

不过，开发者们的这一创造远远不止于改变我们进行交易的方式，它将要改变整个世界。然而，几乎没有人意识到这种改变。

伟大的物理学家爱德华·威滕曾说：

弦理论是一项『属于 21 世纪却不小心降临到了 20 世纪的理论』。

换句话说，物理界根本没有准备好迎接弦理论。

而区块链，则是一项属于 22 世纪却不小心降临到了 21 世纪的技术。

很显然，我们还没有准备好。

什么是区块链？

区块链的核心是一个简洁优雅到令人惊讶的数据结构。本质上，它就是一条链表，每一个区块占据一个节点，并且包含一个指向前一区块的哈希加密。以这样的方式，它创建了一个有效的不可改变的块链以及延伸到原始块的指纹。

接下来，这个系统里面的每一个人在复制块链的同时（并通过重复加密哈希函数验证了合法性），也就缓慢形成了一个有些粗糙的分布式数据库，能够达到防止篡改的目的。

这，就是区块链了。是不是听起来有些不可思议？

当然了，为了实现一个操作性强的协议，要做的可远不止这些——你还需要认证（公钥/私钥加密），共识机制（中本聪通过工作证明或各种证据证明的共识），空间和时间优化（梅克尔树和梅克尔证明），以及一些奇怪的点对点网络的东西。但这是另一个更技术性的文章的主题了。

关键点在于：区块链是一个能够组织复杂系统的开创性新工具。而我们只是开始了解如何将之融入现实世界。

区块链的愿景

如果每个人都处在相同的区块链上，并且所有人都公开分享当前的区块链状态，且保证计算上无法篡改，都同意关于新数据如何实施的规则，那么突然之间，一切都会改变。

你将参与到一个完全分布式的系统中，不需要任何权威机构或个人维护信任。只要系统中有足够的参与者遵循协议（在最原始的情况下，至少 50% 的参与者就够了），那么系统就具有了可证明的安全保证。任何阴谋和权威再也不能审查或者破坏制度。

你甚至可以直接将协议中的正确激励设计到工程中，然后让系统里的每个参与者都执行它们。

有了区块链，许多大规模的组织协调难题都将消失。困扰全球金融基础设施的表决难题、国际汇款难题、保险单难题、监管记录难题等，甚至政府腐败的问题，都将轻而易举地被解决。

通过创造适当的保证制度，还可以纠正一些不利的激励措施，比如消灭掉贪腐的中间人和套利者。总之，这将创造出一个全新的社会，比以往任何时候都能更加透明、有效地协调运转。

然而，大多数人却难以理解这个重大的意义。

我且把话撂在这儿：如果约翰·洛克（英国著名哲学家）知道区块链的话，这无疑将使他写出第三篇关于政府研究的专著。这真的是一个关于如何协调社会的革命性想法。

未来的未来

我知道现在硅谷的大部分人把最重要的技术革新押注在深度学习和 AI 上，我非常支持他们的想法，深度学习一定会推进相关行业的发展，把我们以前只能在科幻小说中幻想的东西变为现实。

但是区块链——将会颠覆整个社会。这将使得乌托邦式的、出现在哲学家空想中的新型社会制度成为可能。

然而，现在看着加密货币的这个世界，你不一定能够认识到这一点。

目前最有可能即将发生的事情有以下两种情况。

其一，无政府主义者和黑客们建立底层协议，他们正在争取创建将成为未来架构的分布式 TCP/IP 协议栈。

其二，牟利者正在快速收敛那些不明真相和对此乐观主义的人群的钱财。

不幸的是，现在人们只关心第二种情况。

区块链之塔

回想一下，1995 年的时候人们在万维网上浏览着小粉丝网站，那时人们在跨越新世纪时想着：”哈，哇。这个奇怪的小玩意儿以后将会变得异常重要吧。”

其实，几年前的区块链恰恰就是那个样子。而现在，区块链已经完全跨越了 Angelfire 的鸿沟。

既然历史已经证明了它初出茅庐时的星光，那么接下来它将一发不可收拾。

你可能听说了最近的 ICO 狂潮，或者比特币和以太坊的暴涨。聪明的人和狡猾的人都已经意识到了区块链的巨大潜力，他们甚至已经迷失在对加密货币潜在未来的猜测和意淫中。

这也是悲哀之处，对区块链的炒作吸引了绝大部分人的注意力，而它真正的价值被淹没在了噪音中。

但这是可预料的，毕竟我们以前到过这样的境况。

当人们最开始意识到互联网的非凡潜力时，大量的钱疯狂投入到 dot com 中，到处是大规模的采购和投机炒作，直到 2001 年狂潮终于坠毁。

这正如现在发生的事情一样，到最终一定会往好的结局发展。

但是当一切尘埃落定，就像 dot com 崩溃时，那些真正举足轻重的公司——微软、亚马逊、谷歌们——将会进来为构建未来做好扎实的工作。

漫漫长路正等待无畏者踏足。

近来我刚刚辞去了在 Airbnb 的工作，尽管 Airbnb 给我提供了良好的环境，我从事了令人着迷的付款诈骗工作一年多，但是我越来越感到区块链在召唤我，我相信这个东西一定会改变世界！

如果你竟然读到了这里，希望你没有被上面蹩脚的语句惹恼。

如你所见，这是我翻译自 Medium 上的一篇文章，心血来潮就吭哧吭哧开始了，没想到我的耐心能支撑我生疏的英文一句一句译下来，当然，这里要感谢友好的『Google 翻译』帮了我不少忙。

记得有个哲人说过：

一个人最糟糕的状态是，不在场和不知道自己不在场。

在互联网中成长起来的我们，似乎天然就存在于互联网这个场中，这两年人工智能的大潮卷来，或多或少也都了解这样一个场的存在了。

但是区块链可能是你缺席的一个大机会，缺席不可怕，可怕的是你不知道你不在场。

这篇文章作者一再强调区块链的非凡意义，但是并没有很友好地讲清楚区块链的含义和应用。假如你刚刚接触区块链并且有着极大的兴趣，我推荐你阅读 GitBook 上的《区块链技术指南》

未来即将癫狂，何不一起狂舞。

语法格式

crontab [-e [UserName]|-l [UserName]|-r [UserName]|-v [UserName]|File ]

crontab -l
列出目前的任务时程表

crontab -e
执行 vi 编辑器设定时程表

crontab cronfile
先编辑文件再命令执行，会创建 cronfile 的一个副本放在 /var/spool/cron 目录中

例子

使用 crontab 执行一个每周四定时发送邮件的任务

crontab -e

任务频率时间设定

指定时间运行

at -f test-cron.sh -v 10:25

以时间格式规定运行

格式：分钟 小时 天 月 星期 命令

10 3 0,6 hello
每周日、周六的3点10分执行 hello 程序
（0，6表示周日和周六）

循环运行

/2 * root /opt/bin/test-cron.sh
没隔2分钟运行一次

cron 的启动

/etc/init.d/crond start
